trackFaceInImage

Introduced in version

public static trackFaceInImage(byte[], int, int, int, ArrayList)

Detect faces in an image in a reduced fashion to speed up detection for anti-spoofing purposes. This variant of the face detection calls returns only face attributes, such as pose estimates and eye and mouth state, such as whether eyes are open or whether there is a smile. No template is ever generated, hence no face can be found that is suitable for recognition. This call uses face landmarks from a previous frame (stored internally) to track possibly multiple faces in a video stream. If a detected face new landmark coordinates fall outside acceptable track quality threshold, full face detection is rerun. Special eye detection and template extraction is removed from the pipeline. All this accomplishes minimal processing per frame, resulting in optimal frame rate, just to get the face attributes. If a face is found that is at least trackable (has the landmark coordinates and pose estimates), then it is returned. All other faces which are rejected are returned in the rejectedFaces array. All faces (except the one returned) are automatically discarded. You may discard the returned face manually, but there is no need as no face returned can actually contain a template. If you require a face to be added to subject templates or to be used for authentication, you must call other variants of the face detection methods.

Parameters

image byte[]
An image object contained in byte array in ARGB format with 8 bits per channel, most likely taken by camera.
width int

Returns

authenticateFace

Introduced in version

public static authenticateFace(String[], String, boolean)

Authenticate face against enrollment templates for a list of given subjects, using a temporary file to pass the image. The largest detected face in the image will be selected for comparison. Results indicate pass/fail decision and confidence measure of decision. Optionally, if smile detection is enabled, results contain a smile decision with confidence. The returned list of results is always the size of the subject list; however, results are sorted by 1) authenticated subjects first 2) confidence value - highest first

Parameters

subjectIds java.lang.String[]
an array of unique identifier strings for subjects, obtained from calling startEnrollment first.
filePath java.lang.String
the path to the image file (jpeg)
antiSpoofing boolean
a flag indicating whether pose compensation should be applied to detected face. If false, pose compensation is applied; if true, no pose correction is applied.

Returns

com.appliedrec.detreclib.authentication.FaceAuthenticationResults[]
An array of objects of type FaceAuthenticationResults, loaded with results of authentication, and a confidence on that decision. Confidence score has a range of 0.5 to 1.0 for either decision. A confidence score of 0.5 means that the decision was on the cusp. A confidence score of 1.0 or close to it means it is a strong decision. For smile, confidence ranges from 0.0 to greater than 50.0 The array is sorted by authentication results and confidence values null is returned if there is no suitable face found for authentication

Set the security level as an integer enum, and save to DB. The security level is global across all enrolled subjects and is persistent across sessions, with a default value of Normal.
